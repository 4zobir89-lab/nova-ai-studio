## 🎯 How to Contribute

### 🐛 Bug Reports

Found a bug? [Open an issue](https://github.com/4zobir89-lab/nova-ai-studio/issues/new?template=bug_report.md) with:

- Clear title and description
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

### ✨ Feature Requests

Have an idea? [Request a feature](https://github.com/4zobir89-lab/nova-ai-studio/issues/new?template=feature_request.md) with:

- Problem statement
- Proposed solution
- Alternatives considered
- Additional context

### 📝 Documentation

- Fix typos
- Improve explanations
- Add examples
- Translate content

### 💻 Code

- Fix bugs
- Implement features
- Add tests
- Refactor code

## 🛠️ Development Setup

### Prerequisites

- Node.js ≥ 20
- npm or yarn
- Git

### Setup

```bash
# 1. Fork the repository
# 2. Clone your fork
git clone https://github.com/YOUR_USERNAME/nova-ai-studio.git
cd nova-ai-studio

# 3. Install dependencies
npm install

# 4. Create a branch
git checkout -b feature/your-feature

# 5. Start development
npm run dev
```

### Available Scripts

| Command | Description |
|---------|-------------|
| `npm run dev` | Start development server |
| `npm run build` | Build for production |
| `npm run test` | Run tests |
| `npm run lint` | Lint code |
| `npm run clean` | Clean build artifacts |

## 🔄 Pull Request Process

### 1. Create Your Branch

```bash
# Feature
git checkout -b feature/amazing-feature

# Bug fix
git checkout -b fix/bug-description

# Documentation
git checkout -b docs/update-readme
```

### 2. Make Your Changes

- Write clean, readable code
- Add tests for new features
- Update documentation if needed
- Follow style guidelines

### 3. Commit Your Changes

```bash
# Use conventional commits
git commit -m "feat: add amazing feature"
git commit -m "fix: resolve bug in orchestrator"
git commit -m "docs: update README"
```

### 4. Push and Create PR

```bash
git push origin feature/amazing-feature
```

Then [create a Pull Request](https://github.com/4zobir89-lab/nova-ai-studio/compare) with:

- Clear title
- Description of changes
- Related issues
- Screenshots (if applicable)

### 5. Code Review

- Address review comments
- Make requested changes
- Get approval from maintainers

### 6. Merge

Once approved, your PR will be merged! 🎉

## 🎨 Style Guidelines

### Code Style

- Use TypeScript
- Follow ESLint rules
- Use Prettier for formatting
- Write meaningful comments

### Commit Messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat:     New feature
fix:      Bug fix
docs:     Documentation
style:    Formatting
refactor: Code restructuring
test:     Adding tests
chore:    Maintenance
```

### Documentation

- Use Markdown
- Include code examples
- Keep it concise
- Add diagrams when helpful

## 🐛 Reporting Bugs

### Before Reporting

1. Check existing [issues](https://github.com/4zobir89-lab/nova-ai-studio/issues)
2. Try to reproduce the bug
3. Check the documentation

### Bug Report Template

```markdown
**Describe the bug**
A clear description of what the bug is.

**To reproduce**
Steps to reproduce the behavior:
1. Go to '...'
2. Click on '...'
3. See error

**Expected behavior**
What you expected to happen.

**Screenshots**
If applicable, add screenshots.

**Environment**
- OS: [e.g., Windows 11]
- Node.js: [e.g., 20.10.0]
- Browser: [e.g., Chrome 120]
```
